QUICK FACTS
================================================================================

Simulator: MuJoCo via robosuite 1.4.1
Conda Environment: qwen-vla (required for all work)
State Representation: 8D end-effector (3D pos + 3D ori + 2D gripper)
Action Representation: 7D (3D pos delta + 3D ori delta + 1D gripper)
HDF5 Full State: 110D (complete MuJoCo internal state)
Control Frequency: 20 Hz
Mean Reversibility Error: 0.060 m (В±1-2 cm, acceptable)
Error Type: Primarily position; minor orientation/gripper
Test Dataset: 148-step "pick_up_soup" demonstration
Validation: Tested on TFDS (143 steps), HDF5 (148 steps)

IMPORTANT NOTES
================================================================================

1. ALWAYS use qwen-vla conda environment
   conda activate qwen-vla

2. ONLY robosuite 1.4.1 works (1.5.2+ removed SingleArmEnv)
   pip install robosuite==1.4.1

3. Inverse action: negate first 6 dims only (keep gripper unchanged)
   inverse_action[:6] *= -1
   inverse_action[6] = 0  # Keep gripper unchanged

4. Limit trajectory length for inverse testing (error accumulates)
   Use first 30-50 steps only

5. Test across multiple demos/tasks (not just one)
   Validate generalization

================================================================================
TROUBLESHOOTING
================================================================================

Issue: ModuleNotFoundError: No module named 'robosuite'
Fix: Activate conda environment: conda activate qwen-vla

Issue: robosuite version conflicts
Fix: Uninstall and reinstall exactly: pip uninstall robosuite -y && pip install robosuite==1.4.1

Issue: LIBERO not found
Fix: Clone and install: cd third_party && git clone https://github.com/Lifelong-Robot-Learning/LIBERO.git LIBERO && cd LIBERO && pip install -e .

Issue: MuJoCo Error: unknown body
Fix: Check robot name in suite.make() - ensure correct robot initialization

Issue: HDF5 file not found
Fix: Verify path and file exists: ls -lh ~/Scene-Graph-VLA/sandbox/*.hdf5

Issue: State reset produces NaN
Fix: Check state is 110D full state, not 8D EEF state; call env.sim.forward()

Issue: Gripper state not changing
Fix: Check gripper command range (-1 to 1 or 0 to 1); may vary by environment
